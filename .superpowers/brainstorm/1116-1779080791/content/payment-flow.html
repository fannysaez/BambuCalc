Tunnel de paiement — Quelle stratégie ?

Comment le client passe de la demande au paiement ?

A

Pré-paiement immédiat (Stripe Checkout)

Client configure → Stripe Checkout → Paiement → Email confirmation

Pour

  • Revenus immédiats, zéro devis manuel
  • Stripe gère PCI, remboursements, webhooks
  • Intégration ~1 semaine

Contre

  • Risque : client paie avant validation admin
  • Nécessite un compte Stripe + Edge Function
  • Prix doit être fixé avant paiement
B

Devis → Validation → Lien Stripe ★ Recommandé

Client configure → Devis PDF envoyé → Tu valides + ajustes → Lien Stripe par email → Paiement

Pour

  • Tu gardes le contrôle du prix final
  • Compatible avec ton flow actuel (devis admin)
  • Stripe Payment Link = 0 code côté serveur
  • Acompte possible (% configurable)

Contre

  • Délai entre demande et paiement
  • Nécessite action manuelle côté admin
C

Préparer le terrain (pas de Stripe maintenant)

Client configure → Demande + email → Tu contactes manuellement

Ajouter maintenant : champ "mode de paiement souhaité" dans le devis (virement, CB, etc.) et structurer le DB pour Stripe plus tard.

Pour

  • Zéro intégration maintenant
  • Architecture prête pour Stripe ensuite

Contre

  • Pas de paiement en ligne
  • Friction maximale pour le client
